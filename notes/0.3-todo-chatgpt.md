# JFTL 0.3 TODO

The remaining work naturally falls into a small number of modules. The goal is to make each module complete and coherent before moving to the next, rather than continuing to add language features.

---

## 1. Runtime Value Model

Finalize the runtime sentinel values and their semantics. The reserved values should be:

- `_missing`
- `_skip`
- `_error`
- `_current`
- `_parent`
- `_top`

`_missing` is a first-class runtime value that propagates through evaluation. It is **not** immediately converted to `null`; conversion happens only during final serialization. `_skip` is not a data value—it is understood only by `foreach` and suppresses output for the current iteration. `null` remains an ordinary JSON value and is always distinct from `_missing`.

Every expression engine should expose `_missing` as falsy while preserving its identity.

---

## 2. Expressions & Interpolation

Finish the interpolation implementation. Support navigation shortcuts (`${.foo}`), explicit engines (`${py=...}`), and reserve `${=...}` for the template's default expression engine (ultimately CEL).

Compile interpolated strings into literal/expression segments for efficient execution. Avoid adding formatting specifiers, nested interpolation, or advanced escaping in this release.

---

## 3. Collection Transforms

Complete the transform library.

Current transforms:

- flatten
- merge
- to_pairs
- from_pairs

Add:

- remove-missing

`remove-missing` should remove direct `_missing` members/elements from the generated collection. Keep the operation shallow; recursive removal can be considered in a future release.

---

## 4. Foreach

Finalize foreach semantics.

Support explicit:

- `var`
- `key`
- `index`

No implicit loop variables.

`_skip` skips an iteration entirely.

`_missing` produces a missing value in the output collection, allowing the caller to preserve or remove it via `remove-missing`.

---

## 5. Compiler & Plugin API

Stabilize the compiler interfaces instead of extending them.

Every expression plugin should implement a consistent contract (`eval`, `eval_bool`, etc.) and receive the same runtime values (`_missing`, `_error`, `_current`, ...).

Document exactly how plugins are expected to represent and propagate runtime sentinel values.

---

## 6. CLI & Data Sources

Complete the CLI.

Support named datasets such as:

```text
--data users=@users.json
--data config=@config.yaml
--data limit=25
```

Complete output modes:

- single document
- stream output
- multi-file output

These modes should be independent of formatting.

## 7. Output Formatting

Support:

- indent
- compact

JSONFold formatter:

- `--jsonfold`
- `--width`
- optional preset/config

Formatting should work uniformly with all output modes.
